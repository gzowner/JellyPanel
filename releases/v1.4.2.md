# JellyPanel v1.4.2 — Fleet Registry Resilience

This cumulative stability release addresses the intermittent empty server registry and false login screen seen while large M3U imports are active.

## Changes

- Loads and renders the server registry before remote Jellyfin, M3U, EPG, library, or Operations requests.
- Adds bounded browser request timeouts.
- Preserves the last confirmed server registry in browser storage during temporary API/database contention.
- Retries the fleet registry automatically every 30 seconds and when the browser tab becomes active.
- Refuses to replace a populated fleet with an unexpected empty response unless a server was explicitly deleted.
- Requires an explicit `/auth/me` 401 before displaying the login screen; timeout/unavailable verification preserves the current screen.
- Makes normal primary-server lookup read-only instead of opening a write transaction and locking the primary row.
- Reuses the active PostgreSQL connection for M3U job progress updates rather than acquiring a second connection inside the reconciliation transaction.
- Expands the Control API PostgreSQL pool from 8 to 16 connections.
- Refreshes only Live TV state after source saves and completed imports rather than reloading the entire panel.
- Keeps database schema 19; no migration is required.

## Upgrade

```bash
cd /root
sha256sum -c galaxytv-jellyfin-v1.4.2.zip.sha256
unzip -o galaxytv-jellyfin-v1.4.2.zip
cd galaxytv-jellyfin-v1.4.2
sudo bash install.sh --mode upgrade
```

After upgrading, press `Ctrl+F5` and verify:

```bash
sudo /opt/galaxytv-jellyfin/scripts/manage.sh health
sudo /opt/galaxytv-jellyfin/scripts/manage.sh servers
sudo /opt/galaxytv-jellyfin/scripts/manage.sh m3u-jobs
```
