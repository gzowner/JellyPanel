# JellyPanel v1.4.3 — M3U Source Rendering Hotfix

This cumulative release fixes the Live TV page error:

```text
server_ids.map is not a function
```

## Fixes

- Normalizes PostgreSQL JSONB `server_ids` into a real JSON array before the M3U API responds.
- Adds browser-side normalization for native arrays, JSON strings, single values, empty arrays, null values, and older cached responses.
- Prevents one malformed source-assignment field from crashing the entire M3U source table.
- Restores rendering for existing sources and imported channels without deleting or re-importing them.
- Preserves assigned-server labels, source editing, category selection, and background-refresh display.
- Exposes the saved Jellyfin tuner/guide synchronization error as a hover tooltip.
- Keeps database schema 19; no migration is required.

## Upgrade

```bash
cd /root
sha256sum -c galaxytv-jellyfin-v1.4.3.zip.sha256
unzip -o galaxytv-jellyfin-v1.4.3.zip
cd galaxytv-jellyfin-v1.4.3
sudo bash install.sh --mode upgrade
```

After upgrading, force-refresh the browser with `Ctrl+F5` and open **Live TV & EPG**.

No media or television services are included. Operators must supply their own authorized content and sources.
